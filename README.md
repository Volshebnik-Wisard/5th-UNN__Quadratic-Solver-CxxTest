# Модульное тестирование методом чёрного ящика на примере квадратного уравнения с CxxTest (Visual Studio / cpp)

Данный репозиторий содержит реализацию и тестирование функции решения квадратного уравнения на C++ с использованием фреймворка CxxTest.

### Цель учебной задачи

Реализовать модульные тесты: научиться выявлять граничные случаи, формулировать тестовые сценарии и добиваться корректной работы функции при любых допустимых входных данных.

## Постановка задачи

Реализовать функцию solve_quadratic(a, b, c, x1, x2), которая решает уравнение:

`a·x² + b·x + c = 0`

### Спецификация возвращаемого значения

| Возвращаемое значение | Смысл |
|:---:|---|
| 2 | Два различных вещественных корня |
| 1 | Один корень (кратный или вырожденный случай) |
| 0 | Нет вещественных корней (отрицательный дискриминант) |
| -1 | Уравнение вырождено (`a = b = c = 0`) |

При отсутствии корней `x1` и `x2` устанавливаются в `NaN`.

### Требования к реализации

- Корректно обрабатывать вырожденные случаи: `a = 0` (линейное), `a = b = 0` (константа), `a = b = c = 0`
- Устойчиво работать при коэффициентах, близких к нулю или очень больших по модулю
- Не допускать переполнения при вычислении дискриминанта

## Тестовые сценарии

Проект содержит 12 тестов, покрывающих все ключевые ситуации:

| Тест | Описание | Пример |
|---|---|---|
| test_equal_roots | Кратный корень | x² + 2x + 1 = 0 → x = −1 |
| test_linear | Линейное уравнение (a = 0) | x + 1 = 0 → x = −1 |
| test_constant | Константа (a = b = 0, c ≠ 0) | Нет решений |
| test_square | Только квадратный член | x² = 0 → x = 0 |
| test_5 | Два различных корня | x² − 1 = 0 → x = ±1 |
| test_zero | Полностью нулевые коэффициенты | Вырожденный случай |
| test_negative_discriminant | Отрицательный дискриминант | x² + x + 1 = 0 → NaN |
| test_negative | Отрицательные коэффициенты | −x² − 2x = 0 |
| test_golden_ratio | Золотое сечение | x² − x − 1 = 0 |
| test_close_to_zero_all | Все коэффициенты ≈ DBL_MIN | Устойчивость к denormals |
| test_close_to_zero | `a` ≈ machine epsilon | Устойчивость к малому `a` |
| test_huge_all | Коэффициенты ≈ √DBL_MAX | Устойчивость к переполнению |

Собрать и запустить проект `Test`. Результаты сохраняются в файл: `TEST-cxxtest.xml`
XML-отчёт совместим с форматом JUnit и читается в большинстве CI-систем (Jenkins, GitLab CI и др.).
